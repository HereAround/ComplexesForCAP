45#F  IsIdenticalObj( <obj1>, <obj2> )  . . . . . . . are two objects identical
46##
47##  <#GAPDoc Label="IsIdenticalObj">
48##  <ManSection>
49##  <Func Name="IsIdenticalObj" Arg='obj1, obj2'/>
50##
51##  <Description>
52##  <Ref Func="IsIdenticalObj"/> tests whether the objects
53##  <A>obj1</A> and <A>obj2</A> are identical (that is they are either
54##  equal immediate objects or are both stored at the same location in 
55##  memory.
56##  <P/>
57##  If two copies of a simple constant object
58##  (see section <Ref Sect="Mutability and Copyability"/>) are created,
59##  it is not defined whether &GAP; will
60##  actually store two equal but non-identical objects, or just a single
61##  object. For mutable objects, however, it is important to know whether
62##  two values refer to identical or non-identical objects, and the
63##  documentation of operations that return mutable values should make
64##  clear whether the values returned are new, or may be identical to
65##  values stored elsewhere.
66##  <P/>
67##  <Example><![CDATA[
68##  gap> IsIdenticalObj( 10^6, 10^6);
69##  true
70##  gap> IsIdenticalObj( 10^30, 10^30);
71##  false
72##  gap> IsIdenticalObj( true, true);
73##  true
74##  ]]></Example>
75##  <P/>
76##  Generally, one may compute with objects but think of the results in
77##  terms of the underlying elements because one is not interested in
78##  locations in memory, data formats or information beyond underlying
79##  equivalence relations. But there are cases where it is important to
80##  distinguish the relations identity and equality.  This is best
81##  illustrated with an example.  (The reader who is not familiar with
82##  lists in &GAP;, in particular element access and assignment, is
83##  referred to Chapter&nbsp;<Ref Chap="Lists"/>.)
84##  <Example><![CDATA[
85##  gap> l1:= [ 1, 2, 3 ];; l2:= [ 1, 2, 3 ];;
86##  gap> l1 = l2;
87##  true
88##  gap> IsIdenticalObj( l1, l2 );
89##  false
90##  gap> l1[3]:= 4;; l1; l2;
91##  [ 1, 2, 4 ]
92##  [ 1, 2, 3 ]
93##  gap> l1 = l2;
94##  false
95##  ]]></Example>
96##  The two lists <C>l1</C> and <C>l2</C> are equal but not identical.
97##  Thus a change in <C>l1</C> does not affect <C>l2</C>.
98##  <Example><![CDATA[
99##  gap> l1:= [ 1, 2, 3 ];; l2:= l1;;
100##  gap> l1 = l2;
101##  true
102##  gap> IsIdenticalObj( l1, l2 );
103##  true
104##  gap> l1[3]:= 4;; l1; l2;
105##  [ 1, 2, 4 ]
106##  [ 1, 2, 4 ]
107##  gap> l1 = l2;
108##  true
109##  ]]></Example>
110##  Here, <C>l1</C> and <C>l2</C> are identical objects,
111##  so changing <C>l1</C> means a change to <C>l2</C> as well.
112##  </Description>
113##  </ManSection>
114##  <#/GAPDoc>
115##
116BIND_GLOBAL( "IsIdenticalObj", IS_IDENTICAL_OBJ );

192##  <Filt Name="IsMutable" Arg='obj' Type='Category'/>
193##
194##  <Description>
195##  tests whether <A>obj</A> is mutable.
196##  <P/>
197##  If an object is mutable then it is also copyable
198##  (see&nbsp;<Ref Func="IsCopyable"/>),
199##  and a <Ref Func="ShallowCopy"/> method should be supplied for it.
200##  Note that <Ref Func="IsMutable"/> must not be implied by another filter,
201##  since otherwise <Ref Func="Immutable"/> would be able to create

204##  <Ref Func="IsMutable"/> is <K>true</K>.
205##  <P/>
206##  In many situations, however, one wants to ensure that objects are
207##  <E>immutable</E>. For example, take the identity of a matrix group.
208##  Since this matrix may be referred to as the identity of the group in
209##  several places, it would be fatal to modify its entries,
210##  or add or unbind rows.
211##  We can obtain an immutable copy of an object with
212##  <Ref Func="Immutable"/>.
213##  </Description>
214##  </ManSection>

282##
283##  <#GAPDoc Label="StructuralCopy">
284##  <ManSection>
285##  <Func Name="StructuralCopy" Arg='obj'/>
286##
287##  <Description>
288##  In a few situations, one wants to make a <E>structural copy</E>
289##  <C>scp</C> of an object <A>obj</A>.
290##  This is defined as follows.
291##  <C>scp</C> and <A>obj</A> are identical if <A>obj</A> is immutable.
292##  Otherwise, <C>scp</C> is a mutable copy of <A>obj</A> such that
293##  each subobject of <C>scp</C> is a structural copy of the corresponding
294##  subobject of <A>obj</A>.
295##  Furthermore, if two subobjects of <A>obj</A> are identical then
296##  also the corresponding subobjects of <C>scp</C> are identical.
297##  <Example><![CDATA[
298##  gap> obj:= [ [ 0, 1 ] ];;
299##  gap> obj[2]:= obj[1];;
300##  gap> obj[3]:= Immutable( obj[1] );;
301##  gap> scp:= StructuralCopy( obj );;
302##  gap> scp = obj; IsIdenticalObj( scp, obj );
303##  true
304##  false
305##  gap> IsIdenticalObj( scp[1], obj[1] );
306##  false
307##  gap> IsIdenticalObj( scp[3], obj[3] );
308##  true
309##  gap> IsIdenticalObj( scp[1], scp[2] );
310##  true
311##  ]]></Example>
312##  <P/>
313##  That both <Ref Func="ShallowCopy"/> and <Ref Func="StructuralCopy"/>
314##  return the argument <A>obj</A> itself if it is not copyable
315##  is consistent with this definition,
316##  since there is no way to change <A>obj</A> by modifying the result of any
317##  of the two functions,
318##  because in fact there is no way to change this result at all.
319##  </Description>
320##  </ManSection>
321##  <#/GAPDoc>
322##

462##  The absolute value gives the minimal length of the result.

464##  characters it is filled with blanks.

466##  if <A>length</A> is negative it is filled on the right.

469##  string (in particular not a part of any other object);
470##  it can be modified safely,
471##  and <Ref Func="MakeImmutable"/> may be safely applied to it.
472##  <Example><![CDATA[
473##  gap> PrintString(123);PrintString([1,2,3]);

475##  "[ 1, 2, 3 ]"
476##  ]]></Example>

478##  characters <C>\&lt;</C> (ASCII 1) and <C>\&gt;</C> (ASCII 2) 
479##  that allow proper line breaks. See <Ref Func="StripLineBreakCharacters"/>
480##  for a function to get rid of these control characters.
481##  </Description>
